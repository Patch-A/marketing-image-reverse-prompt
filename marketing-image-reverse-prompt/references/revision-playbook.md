# Revision Playbook

Use this guide when the user uploads a first-pass generated result and wants a stronger second prompt.

## Common Failure Types

- hero subject too small
- hero subject shape drift
- composition drift
- weak headline hierarchy
- cluttered background
- palette drift
- missing premium feel
- text rendering failure
- missing supporting element

## Correction Pattern

For each revision:
1. name the concrete differences
2. identify the likely cause
3. strengthen the relevant prompt block
4. produce a revised prompt
5. append the change to revision history

## Example Strategies

### Hero Subject Too Small

- increase hero emphasis
- mention centered dominant product scale
- remove competing props

### Headline Hierarchy Too Weak

- explicitly state bold headline placement
- reinforce top positioning
- reduce visual competition around the title area

### Background Too Busy

- simplify background
- reduce decorative props
- specify clean negative space

### Text Rendering Failure

- shorten text if the user allows it
- clarify exact original copy
- reinforce role and placement of the text
- avoid overloading the prompt with too many styling instructions

## Revision History Notes

When a revision appears effective, save:
- failure type
- correction strategy
- revised prompt version
- model target if relevant
